Φ-Protocol: Theoretical Framework & Anti-Overflow Templates
核心论证 (The Thesis)

本项目旨在论证一种基于物理层逻辑闭合的系统代谢模型。通过对底层逻辑的重构，我们提出：系统的安全性不应依赖于算法的复杂性，而应取决于其逻辑边界的物理收敛性。

在 AGI 时代，由于概率性输出带来的逻辑溢出（Logic Overflow）已成为系统性风险。本协议提供了一套防溢出架构模板，用以在硬件底层实现主权闭合。

1. 逻辑理论：忒修斯之船的收敛性 (Theoretical Framework)

我们重新定义了系统的稳定性：

不增不减法则：系统在迭代过程中，其核心逻辑熵值必须保持恒定。

物理主权：通过将决策边界从云端拉回硬件末端（如 ESP32 系列），实现逻辑的不可篡改性。

代谢闭环：系统通过“父子日志”机制进行自我修正，而非依赖外部指令。

2. 防溢出模板 (Anti-Overflow Templates)

本仓库公开部分逻辑隔离模板。这些模板并非完整的执行程序，而是定义了数据流在进入物理执行层前的硬熔断机制：

Boundary-Guard (边缘哨兵)：用于限定子逻辑的反馈波段，防止逻辑递归导致的内存或决策溢出。

Closure-Hook (闭合钩子)：一种硬件级的强制对齐协议，确保每一帧数据处理都在父逻辑的预设轨道内。

[注意]：此处仅提供逻辑骨架与模板定义。核心参数 Φ 已被物理隔离，不在此处进行任何形式的披露。
